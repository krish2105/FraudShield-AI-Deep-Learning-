# Data Folder — FraudShield AI

This folder holds the transaction data used by the notebook and the dashboard.

## Files

| File | Description |
|------|-------------|
| `paysim.csv` | **The REAL dataset (you add this).** Download PaySim and save it here. Not included in the repo because it is large. |
| `sample_transactions.csv` | **Synthetic demo data** generated by `src/make_sample_data.py`. Lets the project run with zero setup. **Demo only.** |
| `README_DATA.md` | This file. |

## ⚠️ Important

`sample_transactions.csv` is **synthetic, for demo / testing only**. Any model
metrics produced from it are **placeholders**. For real results:

1. Download the PaySim dataset (see below).
2. Save it as `data/paysim.csv` with the exact columns listed below.
3. Re-run `notebooks/Payment_Fraud_LSTM_Complete.ipynb`.
4. The notebook will automatically prefer `paysim.csv` over the sample.

## Where to get the real PaySim dataset

PaySim is a public synthetic mobile-money dataset commonly hosted on Kaggle:

> "PaySim — Synthetic Financial Datasets For Fraud Detection"
> https://www.kaggle.com/datasets/ealaxi/paysim1

Download `PS_20174392719_1491204439457_log.csv`, rename it to `paysim.csv`,
and place it in this folder.

## Expected columns

```
step, type, amount, nameOrig, oldbalanceOrg, newbalanceOrig,
nameDest, oldbalanceDest, newbalanceDest, isFraud, isFlaggedFraud
```

| Column | Meaning |
|--------|---------|
| `step` | Time step (1 step = 1 hour of simulation) |
| `type` | PAYMENT, TRANSFER, CASH_OUT, DEBIT, CASH_IN |
| `amount` | Transaction amount |
| `nameOrig` | Sender (origin) customer ID |
| `oldbalanceOrg` | Sender balance before the transaction |
| `newbalanceOrig` | Sender balance after the transaction |
| `nameDest` | Receiver (destination) ID |
| `oldbalanceDest` | Receiver balance before |
| `newbalanceDest` | Receiver balance after |
| `isFraud` | 1 if the transaction is fraud, else 0 (the label) |
| `isFlaggedFraud` | 1 if a naive business rule flagged it |
